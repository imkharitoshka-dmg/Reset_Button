# Reset Button — логика выбора сценария

## Почему не чистый random

Чистый random может повторять один и тот же сценарий несколько раз подряд и игнорировать опыт пользователя. Для Reset Button лучше controlled rotation: локальная rule-based логика, которая учитывает историю, но остаётся простой и предсказуемой.

## Cold start

Если истории по выбранной комбинации состояние × длительность ещё нет, приложение показывает default-сценарий.

Default-сценарий нужен как базовый безопасный вариант:

- он всегда доступен;
- он не зависит от истории;
- он сохраняет понятное поведение при первом запуске.

## Учет результата

После сценария пользователь выбирает:

- помогло;
- частично;
- не помогло.

При следующих выборах для той же комбинации:

- сценарии с результатом `помогло` получают больший приоритет;
- сценарии с результатом `частично` остаются нейтральными или получают небольшой приоритет;
- сценарии с результатом `не помогло` получают меньший приоритет, но не удаляются полностью.

## Избегание повторов

Если в истории есть последний использованный сценарий для этой комбинации, приложение не должно повторять его сразу, если есть доступная альтернатива.

Правило:

1. Найти варианты для состояния и длительности.
2. Исключить последний использованный вариант, если вариантов больше одного.
3. Применить приоритет по результатам.
4. Иногда выбирать редко использованный вариант, чтобы пользователь не зацикливался на одном чек-листе.

## Редко использованные варианты

Чтобы сценарии не повторялись слишком узко, controlled rotation может иногда поднимать приоритет варианта, который давно не показывался или использовался редко.

Это не random ради random, а локальное правило разнообразия.

## Почему RAG не нужен в MVP

RAG не нужен, потому что:

- сценарии короткие и заранее подготовленные;
- весь контент локальный;
- приложение должно работать без интернета;
- рекомендации rule-based достаточно для MVP;
- использование RAG усложнит приватность, качество, тестирование и поддержку.

В MVP выбор сценария должен работать локально через shared_preferences и простые правила.
