# Reset Button

Flutter Android-приложение на русском языке для коротких reset-сценариев на 3, 5 и 10 минут. Приложение помогает сделать короткую паузу, пройти простой чек-лист, отметить результат и сохранить историю локально на устройстве.

## О проекте

Reset Button поддерживает offline-first flow без аккаунта и сервера. Пользователь выбирает текущее состояние, длительность сценария, проходит чек-лист, отмечает результат и при желании добавляет заметку.

Формулировки в приложении спокойные и нейтральные: без медицинских обещаний, без гарантированных результатов и без привязки к полу пользователя.

## Возможности

- 5 кластеров состояний на главном экране.
- 15 пользовательских состояний внутри кластеров.
- Сценарии на 3, 5 и 10 минут.
- Несколько вариантов чек-листов для комбинаций состояние × длительность.
- Быстрый reset на 3 минуты.
- Controlled rotation для выбора варианта сценария.
- Локальная история reset-сессий.
- Сохранение результата: `помогло`, `частично`, `не помогло`.
- Необязательная заметка после reset.
- Компактная локальная аналитика в истории после 5 завершённых сессий.

## Пользовательский flow

```text
главный экран
-> кластер
-> состояние
-> длительность 3 / 5 / 10 минут
-> чек-лист
-> результат
-> заметка
-> история
```

Быстрый reset открывает короткий сценарий на 3 минуты и использует ту же логику controlled rotation, что и обычные сценарии.

## Структура reset-сценариев

Главный экран содержит 5 кластеров:

- Успокоиться
- Восстановиться
- Разгрузить голову
- Сфокусироваться
- Переключиться после общения

Состояния внутри кластеров:

| Кластер | Состояния |
| --- | --- |
| Успокоиться | Меня накрыл стресс; Я тревожусь; Я переживаю из-за будущего |
| Восстановиться | Усталость; Эмоциональное истощение; Мне ничего не хочется |
| Разгрузить голову | У меня хаос в голове; Слишком много задач; Слишком много информации |
| Сфокусироваться | Я не могу сфокусироваться; Я не могу начать; Я распыляюсь |
| Переключиться после общения | Усталость от общения; Был тяжёлый разговор; Мне нужно собраться перед встречей |

Для каждой комбинации состояние × длительность доступны локальные варианты сценариев. Сценарии зашиты в приложение и не требуют интернета.

## Controlled Rotation

Приложение не выбирает сценарий чистым random. Вместо этого используется локальная rule-based логика:

```text
score = defaultPriorityBonus + helpRate + recencyBonus + explorationBonus
```

Правила:

- при первом запуске комбинации показывается default-сценарий;
- если есть история, приложение не повторяет последний `scenarioVariantId`, когда есть альтернатива;
- результат `помогло` повышает вес сценария;
- результат `частично` учитывается слабее;
- результат `не помогло` не усиливает сценарий;
- новые и редко использованные варианты получают `explorationBonus`;
- default-сценарий получает небольшой `defaultPriorityBonus`.

Вся логика работает локально и использует только историю на устройстве.

## История и локальная аналитика

История сохраняется через `shared_preferences`. Для каждой новой сессии сохраняются:

- состояние;
- название сценария;
- длительность;
- результат;
- заметка, если она есть;
- `scenarioVariantId`.

Старые записи без `scenarioVariantId` читаются безопасно. Технический id не показывается пользователю: если вариант найден, история показывает название сценария.

Блок `Что чаще помогало` появляется только после минимум 5 завершённых reset-сессий. На экране истории он остаётся компактным, а подробная статистика открывается по клику:

- первый уровень показывает топ состояний по частоте использования;
- детализация по состоянию показывает сценарии и счётчики `помогло / частично / не помогло`;
- старые названия состояний нормализуются только для аналитики.

- без Firebase;
- без бэкенда;

```bash
flutter run -d emulator-5554
```
